## What are Skills?

[Skills](https://code.claude.com/docs/en/skills) are instruction sets that Claude loads on demand to handle specialized tasks. Instead of Claude improvising how to use the Giza SDK or MCP tools, skills provide exact workflows, tool sequences, and safety guardrails.

This repo ships two skill collections as installable [Claude Code plugins](https://code.claude.com/docs/en/plugins):

- **Developer Skills** (`/giza-*`) -- for building applications with `@gizatech/agent-sdk`
- **Yield Skills** (`/yield-*`) -- for end users managing DeFi positions through conversation

## Installation

<Steps>
  <Step title="Register the marketplace">
    Run this once in any Claude Code session. It tells Claude Code where to find Giza skills:

    ```
    /plugin marketplace add gizatechxyz/agent-sdk
    ```
  </Step>
  <Step title="Install the plugin group(s) you need">
    Pick one or both depending on your use case:

    ```
    # For developers building with the TypeScript SDK
    /plugin install developer-skills@giza-skills

    # For end users managing yield via conversation + MCP tools
    /plugin install yield-skills@giza-skills
    ```
  </Step>
  <Step title="Use any skill by name">
    Mention the skill in your prompt, or type it as a slash command:

    ```
    > Use /yield-start to set up my agent
    > Help me with /giza-manage to deactivate my agent
    ```

    You can also run `/plugin` and open the **Installed** tab to browse installed skills interactively.
  </Step>
</Steps>

<Note>
  **Yield skills prerequisite:** The Giza MCP server must be running in your Claude Code environment. See [MCP Server Quickstart](/mcp-server/quickstart) for setup instructions.
</Note>

### Uninstalling

To remove a plugin group:

```
/plugin uninstall developer-skills@giza-skills
/plugin uninstall yield-skills@giza-skills
```

To remove the marketplace entirely (also uninstalls all its plugins):

```
/plugin marketplace remove giza-skills
```

### Safety Model

Yield skills enforce three confirmation tiers to protect users from accidental actions:

| Tier | Actions | Confirmation |
|------|---------|-------------|
| Read-only | Claim rewards, view portfolio | None |
| Reversible write | Top up, partial withdraw, run optimization | Light confirmation |
| Irreversible | Full withdrawal, deactivate agent | Explicit warning + confirmation |

### MCP Tool Coverage

All 22 MCP tools are covered across the 4 yield skills:

| Category | Tools | Skill |
|----------|-------|-------|
| Wallet | `connect_wallet`, `disconnect_wallet` | yield-start, yield-check, yield-manage |
| Account | `create_smart_account`, `get_smart_account` | yield-start |
| Discovery | `get_tokens`, `get_protocols`, `get_stats`, `get_tvl` | yield, yield-start |
| Lifecycle | `activate_agent`, `deactivate_agent`, `top_up`, `run_agent` | yield-start, yield-manage |
| Portfolio | `get_portfolio`, `get_performance`, `get_apr`, `get_deposits` | yield-start, yield-check, yield-manage |
| Financial | `withdraw`, `get_withdrawal_status`, `get_transactions`, `get_fees` | yield-check, yield-manage |
| Rewards | `claim_rewards` | yield-manage |
| Optimizer | `optimize` | yield-manage |

## Creating Custom Skills

You can create your own skills that build on Giza's tools. Each skill is a folder with a `SKILL.md` file containing YAML frontmatter and markdown instructions:

```markdown
---
name: my-custom-skill
description: What this skill does and when Claude should use it
---

Your instructions here. Claude follows these when the skill is active.
```

Place custom skills in a `skills/` directory and register them in `.claude-plugin/marketplace.json`. See the [Anthropic Skills spec](https://github.com/anthropics/skills/tree/main/spec) for the full format.
