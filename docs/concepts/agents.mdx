## What are Giza Agents?

**Giza Agents** manage capital in smart accounts to maximize yield across DeFi lending protocols. Once activated, they monitor APRs, spot rebalancing opportunities, and execute transactions without manual intervention.

## How Agents Work

### Optimization Cycle

1. **Monitor**: Agent continuously monitors APRs across selected protocols
2. **Analyze**: Calculates optimal allocation considering:
   - Current APRs
   - Gas costs
   - Slippage
   - Protocol limits
   - User constraints
3. **Execute**: If rebalancing improves net returns (after gas), execute transactions
4. **Report**: Update performance metrics and portfolio data
5. **Repeat**: Continue monitoring for next opportunity

## Agent Lifecycle

### States

<Steps>
  <Step title="Inactive">
    Smart account exists but no agent is activated. User funds are not being managed.
  </Step>

  <Step title="Activating">
    Agent initialization in progress. Session keys being granted, initial deposits being allocated. Brief transition state.
  </Step>

  <Step title="Active">
    Agent is running and optimizing. Capital is deployed across protocols, automatic rebalancing occurs.
  </Step>

  <Step title="Deactivating">
    Withdrawal in progress. Agent pulling funds from all protocols, revoking permissions. Brief transition state.
  </Step>

  <Step title="Deactivated">
    Agent stopped, funds returned to user. No active management. Can be reactivated.
  </Step>

  <Step title="Activation Failed">
    Activation encountered an error. Funds remain safe in smart account. User can retry or withdraw.
  </Step>
</Steps>
