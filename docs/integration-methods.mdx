## Overview

Giza has two ways to integrate yield optimization into your application. Which one you pick depends on your technical setup and whether you already have execution infrastructure.

## Comparison

| Feature | Agentic | IaaS (Intelligence) |
|---------|---------|---------------------|
| **What Giza Provides** | Complete solution:<br/>Smart accounts + Execution + Optimization | Optimization intelligence only |
| **What You Manage** | User interface + monitoring |  Execution infrastructure |
| **Transaction Execution** | Automatic by Giza agents | You execute based on optimizer output |
| **Optimization** | Continuous automatic rebalancing | Call optimizer when you want |
| **Learning Curve** | Easy - start in minutes | Moderate - requires execution infrastructure |
| **Flexibility** | Opinionated best practices | Fully customizable |
| **Best For** | Partners without execution infrastructure | Partners with existing execution infrastructure |

## Agentic Integration

### What is it?

The **Agentic approach** uses the Giza Agent SDK to run yield optimization agents. Agents manage capital, rebalance across protocols, and optimize for APR without manual intervention.

**Giza manages everything:**
- Self-custodied smart account creation ([ZeroDev](https://zerodev.app/))
- Session key management
- Transaction execution
- Gas payment
- Continuous optimization
- Protocol rebalancing

**You manage:**
- User interface
- Deposit flow
- Performance display
- Withdrawal requests

### When to Use

<AccordionGroup>
  <Accordion title="You want a complete solution">
    You want yield optimization without building execution infrastructure.
  </Accordion>
  <Accordion title="You don't have smart account infrastructure">
    Giza provides smart accounts, session keys, and execution -- you don't need to build this.
  </Accordion>
  <Accordion title="You need to ship quickly">
    The SDK handles the backend. You wire up the UI and you're done.
  </Accordion>
</AccordionGroup>

## IaaS Integration (Intelligence as a Service)

### What is it?

The **Intelligence as a Service (IaaS)** approach gives you access to Giza's **Optimizer** as a stateless service. You already have your own execution infrastructure (smart accounts, transaction execution, capital management, or even Vaults) and want Giza's optimization to inform your rebalancing decisions.

**Giza provides:**
- Optimization intelligence (optimal allocations)
- APR improvement calculations
- Detailed action plans
- Execution-ready calldata
- Stateless API

**You manage:**
- Your own execution infrastructure
- Rebalancing schedule
- Capital management

### When to Use

<AccordionGroup>
  <Accordion title="You have existing execution infrastructure">
    You already have smart accounts, transaction execution, or Vaults, and just need optimization intelligence.
  </Accordion>
  <Accordion title="You need custom execution logic">
    Build your own rebalancing schedule, risk management, or execution strategies on top of Giza's optimizer.
  </Accordion>
  <Accordion title="You want full control">
    You decide when to rebalance, which suggestions to execute, and how to manage risk. The optimizer just provides intelligence.
  </Accordion>
  <Accordion title="You want the optimization, not the execution">
    Use Giza's optimization engine without handing over execution to agents.
  </Accordion>
</AccordionGroup>
